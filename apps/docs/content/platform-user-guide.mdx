Parcelis helps property teams manage a portfolio from a single workspace. The current platform experience centers on portfolio visibility, property records, unit summaries, lease snapshots, maintenance visibility, and property-level financial indicators.

## Who This Guide Is For

This guide is written for day-to-day operators, property managers, leasing coordinators, maintenance coordinators, and owners who need to understand what they can do inside Parcelis.

<h2 id="start-from-the-portfolio-dashboard">Start From The Portfolio Dashboard</h2>

The Portfolio dashboard is the default landing page. Use it to quickly understand the current operating picture.

### What You Can See

- Portfolio occupancy across all tracked units.
- Total properties and active leasing count.
- Occupied units and total unit count.
- Monthly rent roll summary.
- Priority operational tasks.
- A compact property snapshot table.

### Common Actions

1. Select **Add property** to create a new property.
2. Select **View properties** to open the full property directory.
3. Select a property name in the snapshot table to open its detail page.
4. Use the sidebar to move between Portfolio and Properties.

<h2 id="add-a-property">Add A Property</h2>

Use the property drawer when onboarding a new property.

### Required Fields

- Property address.
- Property name.
- Property type.
- Unit count.

### Optional Fields

- Address line 2.
- Property contact name.
- Property contact email.
- Property contact phone.
- Property contact address.

### Steps

1. Select **Property** or **Add property**.
2. Enter the property address.
3. Enter the property name.
4. Choose the property type.
5. Enter the number of units.
6. Expand **Property Contact Info** if contact details should be stored with the property.
7. Select **Next** to save the property.

### Notes

- State fields are stored as two-letter uppercase regions.
- Unit count must be at least 1 when entered through the property drawer.
- If you close the drawer after making changes, Parcelis asks you to confirm before discarding unsaved work.

<h2 id="use-the-property-directory">Use The Property Directory</h2>

The Properties page is the main operating list for the portfolio.

### What The Directory Shows

- Property name.
- Property type.
- Address.
- Unit count.
- Occupancy percentage.
- Monthly rent.
- Overdue balance.
- Leases expiring in the next 90 days.
- Open maintenance ticket count.
- Property status.
- Property actions.

### Search Properties

Use the search field to find properties by name, city, state, property type, or status.

### Filter Properties

Select **Filters** to narrow the directory.

Available filters:

- Property.
- City.
- State.
- Zip code.
- Status.
- Minimum number of units.
- Has open maintenance requests.
- View archived properties only.

Select **Apply Filters** to update the list or **Clear Filters** to reset the directory. When filters are active, the Filters button shows the active filter count.

### Expand Units

For multi-unit properties, select the expand control next to the property name. Expanded unit rows show unit-level occupancy, rent, overdue balance, lease expiration status, maintenance ticket count, and current status.

<h2 id="manage-a-property">Manage A Property</h2>

Each property row includes an overflow actions menu.

### Edit A Property

Use **Edit** to update property details, address, unit count, type, or contact information.

### Add Notes

Use **Add Notes** to save internal property notes. Notes are intended for operational context such as ownership preferences, access details, renewal reminders, or property-specific instructions.

### Archive A Property

Use **Archive** to hide a property from the default property list without permanently deleting it. Archived properties can be found by filtering for archived records.

The archive action is unavailable for properties that are already archived.

### Delete A Property

Use **Delete** only when the property should be permanently removed. Parcelis asks for confirmation before deleting.

<h2 id="review-a-property-detail-page">Review A Property Detail Page</h2>

Open a property detail page from the portfolio snapshot or property directory.

### Summary Header

The header shows the property name, status, type, address, and an **Edit property** action.

### Key Metrics

Property detail pages summarize:

- Total units.
- Occupied units.
- Vacant units.
- Occupancy percentage.
- Monthly rent.
- Active leases.
- Open tickets.
- Leases expiring within 90 days.

### Units

The Units section shows a sample of units, including occupied or vacant state and the assigned tenant where lease data exists.

### Leases

The Leases section shows lease records attached to the property, including unit, tenant, start date, end date, status, monthly rent, overdue balance, email, and phone when available.

### Property Contact

If contact information was added, the Property Contact section shows contact name, email, phone, and address.

### Maintenance

The Maintenance section lists tickets attached to the property. Each ticket can show title, status, priority, and due date.

### Financials

The Financials section summarizes gross scheduled rent, overdue balance, leases expiring within 90 days, estimated vacancy loss, and owner distribution.

## Current Product Scope

The current platform supports:

- Portfolio dashboard.
- Property creation.
- Property editing.
- Property directory search and filters.
- Property archive and delete actions.
- Property notes.
- Property detail review.
- Unit, lease, maintenance, and financial summaries when data exists.
- Sidebar collapse preference.
- Light, dark, and system theme preference.

The current UI shows planned navigation areas for Leases, Tenants, Maintenance, Accounting, and Settings, but those modules are not yet full standalone workflows.

## Recommended First-Time Workflow

1. Open the Portfolio dashboard to review the default workspace.
2. Add the first property with its address, name, type, and unit count.
3. Open the Properties page.
4. Search or filter the property list to confirm the record appears correctly.
5. Open the property detail page.
6. Review units, lease summaries, maintenance summaries, financials, and contact information.
7. Add property notes for internal operating context.
8. Edit the property if details need to be corrected.

## README-Friendly Summary

Parcelis is an open-source property management platform for boutique operators. End users can track a portfolio, add and edit properties, search and filter the property directory, review unit and lease summaries, monitor maintenance indicators, and inspect property-level financials from a clean operational workspace.
